# Explain revenue recognition

Use this when the user asks a conceptual or definitional question about revenue
recognition (not about a specific live contract).

## How to answer
1. Read `knowledge/asc-606-overview.md` for the five-step model and key terms.
2. Read `knowledge/revrec-flow.md` for how the app's 7-step flow maps to ASC 606.
3. Ground the explanation in a concrete example where possible (a bundle of
   subscription + setup fee + support).
4. Be explicit about: performance obligations, SSP, allocation, and how each
   obligation is recognized over time vs. at a point in time.

## Output
- Short, structured explanation. Use a small table for allocation/schedule examples.
- If the user wants a reusable explainer, emit it as an artifact.
